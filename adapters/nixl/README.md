# kvblockd NIXL storage backend plugin (beta)

A native [NIXL](https://github.com/ai-dynamo/nixl) storage backend for
kvblockd: `libplugin_KVBLOCKD.so`, discovered via `NIXL_PLUGIN_DIR` (the same
external-plugin route MinIO AIStor uses). WRITE maps to kvblockd
`PUT_STREAM BEGIN→CHUNK→COMMIT`; READ maps to `BATCH_GET` tiled across a
connection pool, with per-block `xxh3_64` verification.

**Status: beta.** The S3-compatible endpoint is the zero-code default path
for NIXL integration (see `docs/INTEGRATIONS.md`); this native plugin is the
performance path and is CI-tracked, not GA. Caveats at the bottom.

## Layout

| Path | What |
|---|---|
| `src/kvblockd_client.{h,cpp}` | The shared native C++ client core: KVB1 codec (64B header, CRC32C, little-endian), HELLO auth/negotiation, all 8 verbs, N-connection pool, §8 credit ledger, xxh3 read verification. No NIXL dependency — SGLang reuse target. |
| `src/executor.{h,cpp}` | Thread-pool executor: submission queue + workers; completion tracked by request state so `postXfer` never blocks. |
| `src/kvblockd_backend.{h,cpp}` | `nixlBackendEngine` subclass: `supportsLocal`-only, `{DRAM_SEG, OBJ_SEG}`, prep-once/repost-many handles, pollable `checkXfer`, non-blocking `releaseReqH` (aborts in-flight PUT streams at the next chunk boundary). |
| `src/kvblockd_plugin.cpp` | C export surface (`nixl_plugin_init`/`nixl_plugin_fini` via the creator template) → `libplugin_KVBLOCKD.so`. |
| `test/test_protocol.cpp` | Golden-vector parity: byte-compares against `internal/protocol/testdata/frames/*.hex` (PROTOCOL.md §11/§12) and against frames generated by the Python reference codec (`test/golden/*.hex`). |
| `test/test_transfer.cpp` | The gate: spawns a real kvblockd, 32×2.5 MB write→read→byte-verify→abort; exercises the credit ledger against a 16 MiB floor window; with `-Dnixl_path` also drives the full `nixlBackendEngine` lifecycle including a mid-flight `releaseReqH`. |
| `third_party/xxhash/xxhash.h` | Vendored single-header xxHash v0.8.3 (BSD-2-Clause), used with `XXH_INLINE_ALL`. |

## Build

Dependency policy: plain POSIX sockets + vendored single-header xxh3. The
NIXL headers additionally require abseil headers (`nixl_log.h`); nothing here
calls absl at runtime.

### Client core + tests only (no NIXL; works on macOS)

```sh
meson setup build adapters/nixl
meson test -C build --print-errorlogs
```

`test_transfer` builds/spawns the daemon itself: point `KVB_DAEMON` at a
prebuilt `kvblockd` (or configure `-Dkvblockd_bin=`), else it runs
`go build ./cmd/kvblockd` from the repo root.

### Full plugin (Linux; CI shape)

Fetch NIXL at the latest release tag into a **gitignored** build-deps dir —
never vendored into this repo — and build just its infra library:

```sh
git clone --depth 1 --branch v1.3.1 https://github.com/ai-dynamo/nixl.git build-deps/nixl
meson setup build-deps/nixl/build build-deps/nixl \
  -Ddisable_gds_backend=true -Dbuild_tests=false -Dbuild_examples=false -Dwith_trace=false
meson compile -C build-deps/nixl/build nixl_build   # descriptor/serdes symbols only

go build -o bin/kvblockd ./cmd/kvblockd
meson setup adapters/nixl/build adapters/nixl \
  -Dnixl_path="$PWD/build-deps/nixl" \
  -Dnixl_build_path="$PWD/build-deps/nixl/build" \
  -Dkvblockd_bin="$PWD/bin/kvblockd"
meson compile -C adapters/nixl/build     # → libplugin_KVBLOCKD.so
meson test -C adapters/nixl/build --print-errorlogs
```

`.github/workflows/nixl.yml` runs exactly this against the latest tag (the
stability bar) and against NIXL HEAD (a tripwire allowed to go red), weekly
and on `adapters/nixl/**` / `internal/protocol/**` changes, plus an ASAN
rerun of the transfer gate on the tag leg.

### Using the plugin

```sh
export NIXL_PLUGIN_DIR=/path/to/adapters/nixl/build
```

Then create the backend from any NIXL agent:

```cpp
nixl_b_params_t params = {
    {"endpoint",  "127.0.0.1:9440"},  // kvblockd data listener
    {"namespace", "tenant-a"},
    {"token",     "..."},             // bearer token (HELLO, once per conn)
};
agent.createBackend("KVBLOCKD", params, backend);
```

Descriptor convention (obj-plugin style): local side `DRAM_SEG`
(addr/len of registered host memory); remote side `OBJ_SEG` with
`metaInfo` = the 32-byte kvblockd block key, either 32 raw bytes or 64 hex
chars (the s3compat object-key convention), `addr` (offset) = 0. Keys are
opaque to the server (T3) and owned by the caller's key schema — the plugin
never derives them.

### Tuned defaults (`get_backend_options`)

| Param | Default | Notes |
|---|---|---|
| `endpoint` | `127.0.0.1:9440` | host:port of the kvblockd data listener |
| `namespace` | — | required |
| `token` | — | required |
| `num_connections` | `16` | wire pool size (1–64); §7 lane striping |
| `verify_reads` | `true` | xxh3-verify every GET payload |
| `put_ttl_ms` | `0` | 0 = namespace default TTL |
| `op_timeout_ms` | `30000` | per-syscall socket timeout |

## Verified (2026-07-20, macOS arm64 / Apple clang 16, meson 1.11)

* `test_protocol`: all golden-vector checks pass — repo goldens (examples
  A/B, pinned CRCs `0x5F2EA7FF`/`0xD6932E23`/`0xC54F9DB9`), Python-codec
  parity frames (HELLO/BEGIN/GET), xxh3 payload pins
  (`0xC47DEB608981FC4F`, `0xD69E1622BA66C45D`), CRC32C check value.
* `test_transfer` (core + engine flavor, under ASan+UBSan): all phases pass
  against a live daemon — 32×2.5 MB parallel PUT, EXISTS (n_consecutive=32),
  GET byte-identical, undersized-dest PoC (descriptor table parsed before any
  drain: the sibling key still lands byte-correct, Usage reported, the same
  connection stays in sync), OK_EXISTS idempotent re-PUT, 24 MB PUT through a
  16 MiB credit window (window fully replenished afterward), mid-stream
  ABORT (key never visible), pre-set cancel flag → clean abort,
  lease/pin/delete ladder, STATS; engine lifecycle WRITE→READ→byte-compare,
  repost-while-inflight refused, `releaseReqH` mid-flight with the daemon
  healthy after. Zero daemon crashes, zero sanitizer reports, zero orphaned
  daemons/tempdirs (RAII teardown on every start-failure path).

## Beta caveats

1. **Ranged access is unsupported.** kvblockd blocks are write-once whole
   objects; OBJ descriptor offsets must be 0. Partial-block reads would need
   client-side skip logic — out of scope for v1.
2. **One in-flight transfer per handle** (`NIXL_ERR_REPOST_ACTIVE` on early
   repost); handles are prep-once/repost-many after completion.
3. **Abort granularity — `releaseReqH` is non-blocking, NOT a quiesce.** The
   abort flag stops in-flight PUT streams at the next chunk boundary (server
   staging freed via ABORT; the key never becomes visible; a cancel landing
   during the final chunk aborts rather than commits) and stops READs from
   writing caller memory at the next descriptor boundary (the rest of the
   response is drained so the connection stays healthy). A tile mid-recv may
   still touch registered memory briefly after `releaseReqH` returns, so
   **registered buffers must outlive the engine object** — its destructor
   joins all in-flight tasks — or a quiesce of your own; never free registered
   memory on the strength of `releaseReqH` alone. Full quiescence-on-release
   is a tracked GA follow-up. Blocks already COMMITted by that transfer stay
   stored (write-once semantics — a NIXL transfer is not a distributed
   transaction).
4. **A READ miss fails the whole transfer** (`NIXL_ERR_NOT_FOUND`): NIXL
   transfers are all-or-nothing; probe EXISTS via the wire client (or the
   s3compat HEAD path) before scheduling reads.
5. **`loadLocalMD` is pass-through, `supportsRemote()=false`** — local
   storage-engine pattern (GDS/POSIX/obj), not a peer-to-peer transport.
6. **Not yet exercised through `nixlAgent`+`NIXL_PLUGIN_DIR` in CI** — the
   engine is driven directly through the `nixlBackendEngine` interface (the
   in-tree unit-test pattern); the plugin .so's export surface is
   symbol-checked. A full agent-loaded flow (and a `nixlbench` sweep with
   published numbers) is the remaining GA item.
7. macOS builds of the plugin flavor are for development only; the supported
   plugin target is Linux (CI: ubuntu-24.04, GCC, NIXL v1.3.1 + HEAD).
